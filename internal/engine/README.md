# `internal/engine`

`internal/engine` 패키지는 무한 서버의 실질적인 게임 핵심 로직(Game Loop) 및 커맨드 시스템을 담당합니다.

## 디렉토리 구조 및 역할

- **`game/` (게임 루프 및 세션)**
  - `loop.go`: 멀티스레드 환경에서 동작하는 핵심 게임 루프. 활성 접속 세션(`ActiveSessions`)을 추적하고, 커맨드 파싱과 디스패치를 수행합니다.
  - 플레이어의 체력 자연 회복, 몹 스폰(Regen), 틱(Tick) 단위의 글로벌 이벤트 갱신을 담당합니다.

- **`command/` (명령어 디스패치)**
  - 사용자가 입력한 문자열을 파싱하고 실행하는 디스패처.
  - `table/` 에는 '이동(Go)', '대화(Say)', '가족(Family)', '죽이기(Kill)' 등 수십 종의 실제 인게임 액션 커맨드가 정의되어 있습니다.
  - 디스패치는 인터페이스 기반의 `CommandSpec`을 이용하여 권한(DM인지 일반 유저인지) 및 조건 검사를 거친 후 실행됩니다.
  - Prometheus 메트릭(`CommandsProcessed`)과 연동되어 어떤 명령어가 서버에서 가장 빈번하게 사용되는지 통계를 제공합니다.

- **`legacy/`**
  - C언어에서 포팅된 텍스트 처리, 랜덤 시드 생성 등 레거시 로직의 원본 형태를 보존하면서 Go 친화적인 형태로 컨버전된 유틸리티 코드들이 위치합니다.

## 동작 방식 요약

1. 게임 루프(`game.Loop`)는 `select` 문을 기반으로 동작하며, 1초 단위 등 특정 틱(Tick)마다 트리거되는 타이머가 있습니다.
2. 유저가 텍스트 입력 후 엔터를 누르면 패킷 큐에 들어가고, 파서가 단어를 토큰 단위로 분할합니다.
3. 분할된 토큰을 `command.Dispatcher`가 분석해 매칭되는 액션 객체의 `Execute()` 메서드를 호출합니다.
4. 실행 결과는 `session` 버퍼를 통해 텔넷 접속자에게 텍스트 형태로 리턴됩니다.
