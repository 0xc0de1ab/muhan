# 무한대전 초보자 플레이 매뉴얼

이 문서는 `muhan-server`와 `muhan-cli` tmux 세션으로 실제 접속해서 확인한 초보자용 플레이 기록이다. 기준 계정은 `인제로`, 비밀번호는 `1234`이다.

## 1. 접속과 로그인

클라이언트 세션에 이미 접속되어 있으면 화면 맨 아래의 `>` 프롬프트에서 바로 이어서 입력하면 된다.

처음 접속하면 다음 순서로 진행된다.

1. 서버에 연결하면 `당신의 이름은 무엇입니까?`가 나온다.
2. `인제로`를 입력한다.
3. `암호를 넣어 주십시요:`가 나오면 `1234`를 입력한다.
4. 로그인에 성공하면 `무한에 접속했습니다.`와 로그인 공지가 나오고, `[엔터]` 안내에서 Enter를 누르면 `>` 프롬프트가 나온다.

`>`는 명령을 입력할 수 있다는 뜻이다. 명령을 입력하고 Enter를 누르면 결과가 출력되고 다시 `>`가 나온다.

## 2. 첫 화면 읽기

로그인 공지를 넘긴 뒤 `봐`를 입력하면 시작 방인 `통계 무한 광장`이 보인다.

중요하게 볼 부분은 세 가지다.

- 방 이름: `통계 무한 광장`
- 방 설명: 생명의 나무, 게시판 등 현재 장소의 분위기와 단서가 나온다.
- 출구 목록: `[ 출구 : 나가는길, 백화점, 무한청사, 밑, 북, 패거리전쟁터, 지존관, 서, 동, 남 ]`

출구 목록에 있는 단어 중 방향어는 그대로 입력하면 이동한다. 예를 들어 `북`, `서`, `동`, `남`은 바로 이동했다. 이름이 붙은 출구는 `백화점 가`처럼 `출구이름 가` 형태로 입력해야 이동하는 경우가 있었다.

## 3. 도움말 확인

가장 먼저 입력해 볼 명령은 다음이다.

```text
도움말
```

실제 반응은 긴 명령 목록이다. 이동, 통신, 정보, 행동, 직업별 기술이 나뉘어 출력된다. 마지막에는 다음 안내가 나온다.

```text
"<명령> 도움"이라고 치면 자세한 설명이 나옵니다.
```

예를 들어 말하기 명령은 다음처럼 확인했다.

```text
말 도움
```

출력 요약:

```text
도움말:
        <문장> 말
        <문장> "
        <문장> '
        <문장> ?
        <문장> .

'말해'는 같은 방에 있는 사람들에게 의사를 전달합니다.
```

주의할 점: 도움말에 보이는 일부 명령은 아직 실제 동작과 차이가 있다. 아래 문제 목록에 따로 정리했다.

## 4. 현재 방 다시 보기

방 설명을 다시 보고 싶으면 다음을 입력한다.

```text
봐
```

`통계 무한 광장`에서는 방 이름, ASCII 그림, 설명, 출구, NPC가 다시 출력됐다.

처음에는 길을 외우려고 하지 말고, 이동한 뒤마다 `봐`를 입력해서 현재 위치와 출구를 확인하는 방식이 안전하다.

## 5. 보이는 물건과 NPC 살펴보기

이 게임은 일부 명령이 일반적인 `명령 대상` 어순이 아니라 `대상 명령` 어순으로 동작했다.

광장에 보이는 사물함을 보려면 다음처럼 입력한다.

```text
사물함 봐
```

실제 출력:

```text
물건을 몇개 넣을수 있는 사물함입니다.
내용물: (x2) 여포의 단도
```

게시판은 다음처럼 볼 수 있었다.

```text
게시판 봐
```

실제 출력:

```text
나무로 만들어진 게시판입니다.
```

게시판 글 목록은 `게시판` 또는 `게시판 읽어`로 나온다.

```text
게시판
게시판 읽어
```

출력 요약:

```text
번호 올린이       날짜  줄수 조회 제목
---------------------------------------------------------------
 142 유리           01/03    7    8 [잡담] Y2K 게시판 해결...
 ...
```

글 본문은 번호를 붙여 읽는다. 두 어순 모두 동작한다.

```text
142 게시판
게시판 142
게시판 142 읽어
```

새 글은 `써`로 시작한다. 제목을 입력한 뒤 본문을 한 줄씩 쓰고, 줄 처음에 `.`을 입력하면 등록된다. 중간에 취소하려면 줄 처음에 `!!`를 입력한다.

```text
써
제목 입력
본문 첫 줄
본문 둘째 줄
.
```

내가 쓴 글은 다음 형식으로 삭제할 수 있다. 실제 본문 파일을 지우는 방식이 아니라 목록/읽기에서 숨기는 방식이다.

```text
게시판 143 글삭제
```

NPC 계석치무도 같은 방식으로 볼 수 있었다.

```text
계석치무 봐
```

실제 출력:

```text
생명의 나무를 지키는 현자인 계석치무가 명상에 잠겨 있습니다
```

현재 포팅 서버는 초보자 편의를 위해 명령어를 앞에 두는 입력도 받아준다.

```text
봐 사물함
봐 게시판
봐 계석치무
```

위 입력도 각각 `사물함 봐`, `게시판 봐`, `계석치무 봐`와 같은 의미로 동작한다.

## 6. 소지품 보기

내가 가진 물건을 보려면 다음을 입력한다.

```text
소지
```

실제 계정에는 매우 많은 물건이 들어 있었다. 출력은 `소지품:` 목록이 길게 이어지고, 마지막에 `장비:`가 나온다.

확인 당시 장비 상태는 다음처럼 나왔다.

```text
장비:
없음
```

초보자 입장에서는 `소지` 출력이 길 수 있으므로, 현재 가진 물건 목록과 장비 상태를 함께 보여주는 명령이라고 이해하면 된다. 마법감지 상태이면 소지품 목록의 마법 물건에 `(+n)` 또는 `(주문)` 같은 표시가 붙을 수 있다. 장비 목록은 기존 이름 중심으로 표시된다.

기본 상태는 다음 명령으로 따로 확인할 수 있다.

```text
정보
[엔터]
점수
장비
시간
어디
상태
```

`정보`는 이름, 칭호, 레벨, 종족, 직업, 능력치, 체력, 도력, 경험치, 돈,
소지품 무게, 무기사용능력을 보여준 뒤 입력을 기다린다. 여기서 엔터를
누르면 주술 계열, 배운 주문, 현재 주문, 임무, 무적수련 정보가 이어서
나오고, `.`을 입력하면 중단된다. `점수`는 기본 체력/도력/방어력/돈
요약을 보여준다. `장비`는 현재 착용하거나 무장한 물건을 보여준다.

플레이어 설정은 대상 없이 `설정`을 입력하면 현재 상태와 간단한 도움말을
보여준다. `잡담듣기 설정`, `도망수치#5 설정`, `패거리귀환 설정`처럼
대상을 붙이면 런타임 상태를 바꾼다. `해제`도 대상 없이 쓰면 도움말을
보여주지만, 대상을 붙이면 설정을 되돌릴 수 있다. 초보자는 먼저 `설정`
단독으로 현재 상태만 확인하는 편이 안전하다.

가진 물건의 세부 수치를 보려면 `비교`를 쓴다. 도둑 계열이나 관리자급 캐릭터는 `감정`으로 사용 회수, 종류, 타격치, 특성까지 볼 수 있다.

```text
목검 비교
목검 감정
감정 목검
```

`비교`는 무기와 방어구의 사용 가능 레벨을 알려준다. `감정`은 도둑이 아니면 `도둑만 물건을 감정할수 있습니다.`가 나온다. 마법감지 상태인 도둑이나 관리자급 캐릭터가 `감정`을 쓰면 마법 힘과 남은 충전 같은 추가 정보가 보일 수 있다.

## 7. 접속자 확인

현재 접속자를 확인하려면 다음을 입력한다.

```text
누구
```

실제 출력:

```text
접속자:
 - 인제로
```

혼자 접속한 상태라면 자기 이름만 보인다.

## 8. 말하기

말하기는 legacy 방식으로 다음처럼 문장을 앞에 둘 수 있다.

```text
안녕하세요 말
```

실제 출력:

```text
당신이 말합니다: 안녕하세요
```

큰따옴표 약식도 동작했다.

```text
안녕하세요 "
```

실제 출력:

```text
당신이 말합니다: 안녕하세요
```

현재 포팅 서버는 다음처럼 명령어를 앞에 두는 입력도 지원한다.

```text
말 안녕하세요
```

감정 표현은 말하기와 별도 명령이다. 혼자 쓸 수도 있고, 같은 방의 플레이어나 NPC 이름을 앞이나 뒤에 붙일 수도 있다.

```text
미소
계석치무 안녕
안녕 계석치무
환하게 웃는다 표현
도와줘 외쳐
Bob 귓속말 이야기
잘 들려 대답
모두 안녕 잡담
```

`표현`은 같은 방에 자유 행동 문장을 보여주고, `외쳐`는 같은 방과 인접 방에 외침을 전달한다. `이야기`는 접속 중인 특정 사용자에게만 보내고, `대답`은 마지막으로 나에게 이야기한 사용자에게 답한다. `잡담`은 접속자 전체에게 보낸다.

NPC와 이야기할 때는 NPC 이름 뒤에 `대화`를 붙인다. 특정 주제를 물어보려면 NPC 이름, 주제, `대화` 순서로 입력한다.

```text
계석치무 대화
계석치무 임무 대화
```

`계석치무 대화`는 환영 대사를 출력했고, `계석치무 임무 대화`는 타버린 두루마기와 치료 안내를 출력했다.

## 9. 같이 다니기와 그룹말

다른 접속자와 함께 움직이고 싶으면 같은 방에서 `따라`를 쓴다.

```text
Bob 따라
```

성공하면 내가 Bob을 따라다니는 상태가 된다. 이후 Bob이 이동하면 나도 같은 출구로 함께 이동한다. 현재 그룹 상태는 `그룹` 또는 `무리`로 확인한다.

```text
그룹
```

그룹 안에만 말을 보내려면 `그룹말`, `무리말`, `=`를 쓴다.

```text
모두 준비됐나요 그룹말
모두 준비됐나요 =
```

따라가기를 멈추려면 `나 따라` 또는 `내보내`를 쓴다. 내가 리더일 때 특정 사람을 그룹에서 빼려면 `<이름> 내보내`를 쓴다.

```text
나 따라
내보내
Bob 내보내
```

`잡담`은 접속자 전체에게 보이고, `이야기`는 한 사람에게만 보인다. `그룹말`은 나와 같이 다니는 그룹 안에서만 보인다.

그룹 상태에서 몬스터를 처치하면 마지막 공격자가 속한 그룹의 런타임
스냅샷이 보상 계산에 들어간다. 기본 경험치는 각 플레이어가 실제로
누적한 피해 비율대로 받고, 그룹 보너스는 그 위에 더해진다. 리더는 보상
경험치의 20% 단위로 `팔로워 수 + 1`만큼, 팔로워는 보상 경험치의 25%
단위로 팔로워 수만큼 추가 보너스를 받는다. 보너스 계산은 몬스터 경험치와
피해 비율을 기준으로 제한되며, DM 투명 상태인 팔로워는 보너스 인원수에
들어가지 않는다.

## 10. 패거리 통신

패거리에 속한 캐릭터는 `패거리누구`로 현재 접속 중인 같은 패거리원을 확인한다.

```text
패거리누구
```

같은 패거리 안에만 말을 보내려면 `패거리말` 또는 `]`를 쓴다.

```text
전원 준비 패거리말
전원 준비 ]
```

패거리에 속하지 않은 상태에서는 `당신은 패거리에 속해있지 않습니다.`가 나온다. 현재 포팅 서버는 기존 플레이어 파일에서 옮겨온 `familyFlag`, `familyID` 기준으로 패거리 소속을 판단한다. `player/family/family_list`와 `family_member_N`도 읽어 패거리 이름, 보스명, 구성원 기록을 UTF-8 모델에 보존한다.

패거리 전쟁은 현재 최소 플레이 명령 경로가 연결되어 있다. 패거리 보스
플래그가 있는 캐릭터가 상대 패거리 이름이나 번호를 붙여 `전쟁선포`를
쓰면, 상대 패거리 보스가 접속 중인지 확인한 뒤 요청, 취소, 수락 상태를
바꾸고 접속자에게 알린다.

```text
패거리2 전쟁선포
전쟁선포 패거리2
```

전쟁 점수는 PvP 사망/RecordFamilyWarKill 경로에서 기록되고, 보스 사망 또는
EndActiveFamilyWar 경로는 최종 점수와 승자 보상 방송을 만든다. 다만
전쟁터 제한, 운영자 개입, 전쟁 상태 HTTP API는 아직 별도 운영 기능으로
보지 않는 편이 안전하다.
`special=3`, `special=SP_WAR`, `SP_WAR=1`
처럼 패거리 전쟁용 특수 물건으로 표시된 아이템을 `사용`하면 일반 주문서,
완드, 물약 경로로 넘어가지 않고 C `special_obj` 기본값인 `아무것도 없습니다.`가 출력된다.
`모든패거리`는 현재 패거리 이름, 문주, 가입축하금, 확인 가능한 패거리
금고 값을 보여준다. `패거리원`은 로드된 `family_member_N` 구성원 목록을
보여주고, `패거리공지`는 내 패거리의 공지 파일을 읽는다. 패거리 은행은
패거리 금고/창고 방에서 `기부`, `인출`, `기증품`, `<물건> 기증`, `<물건> 반출`의 최소
경로가 연결되어 있다. `패거리가입`, `가입허가`, `가입취소`,
`패거리탈퇴`, `패거리추방`도 현재 접속자와 런타임 상태 기준으로 최소
동작한다. 패거리 멤버 파일 갱신과 직업 변경 rewrite 경로는 구현되어
있지만, 초보자 smoke에서는 `--copy-root`로 뜬 서버에서 `패거리원` 전후와
재시작 후 표시를 비교해서 확인한다. 가입 보조금/금고 차감, 탈퇴 비용,
오프라인 사용자 추방은 아직 완성된 플레이 기능으로 보지 않는 편이 안전하다.
패거리 공지는 `패거리공지 a`로 추가하고 `.`으로 저장을 끝내며,
`패거리공지 d`는 문주가 legacy 공지 파일과 JSON sidecar를 함께 삭제한다.

## 11. 기본 이동 연습

광장에서 `북`을 입력하면 `순간이동을 위한 방`으로 이동했다.

```text
북
```

출력 요약:

```text
순간이동을 위한 방

자 티켓 주세요~ 천황마신님이 티켓 검사를 하고 계십니다.
[ 출구 : 남, 순간이동룸 ]
```

다시 광장으로 돌아가려면 출구에 보이는 반대 방향인 `남`을 입력한다.

```text
남
```

광장에서 `동`을 입력하면 `건의 & 불만 & 기타 등등.. 게시판 방`으로 이동했다.

```text
동
```

이 방의 출구는 `[ 출구 : 서 ]`였으므로, `서`를 입력하면 광장으로 돌아왔다.

광장에서 `서`를 입력하면 `아파트 가는 길`로 이동했다.

```text
서
```

이곳의 출구는 `[ 출구 : 서, 동 ]`이고, `동`을 입력하면 광장으로 돌아왔다.

광장에서 `남`을 입력하면 `패거리 거리`로 이동했다.

```text
남
```

출구는 `[ 출구 : 북, 은형문, 무영문, 사랑회, 하리파 ]`였고, `북`으로 광장에 돌아왔다.

## 12. 이름 있는 출구로 이동하기

광장 출구 목록에는 `백화점`처럼 방향이 아닌 이름도 나온다. 이 경우 이름만 입력하면 이동하지 않았다.

```text
백화점
```

실제 출력:

```text
무슨 말인지 모르겠습니다.
```

다음처럼 `가`를 붙이면 이동했다.

```text
백화점 가
```

출력 요약:

```text
백화점 로비

백화점의 로비이다.
[ 출구 : 위, 밑, 기념품점, 귀금속점, 장신구점, 고급매점, 밖 ]
```

백화점 안에서도 이름 있는 출구는 같은 방식으로 이동할 수 있었다.

```text
기념품점 가
```

출력 요약:

```text
기념품점

여러가지 물건을 살수 있는 기념품점이다.
[ 출구 : 밖 ]
기념품을 팔아 생계를 이어가는 묘고부가 서 있습니다.
```

기념품점에서 `밖`을 입력하면 백화점 로비로 돌아오고, 백화점 로비에서 다시 `밖`을 입력하면 광장으로 돌아온다.

## 13. 잠긴 문 따기

잠긴 출구는 보통 `열어`만으로 열리지 않는다. 도둑 계열 캐릭터는 `따`로 잠금을 풀 수 있다.

```text
서 따
```

성공하면 `당신은 문을 따는데 성공했습니다.`가 나오고, 잠금만 풀린다. 닫힌 문은 그대로 닫혀 있을 수 있으므로 이후 `서 열어` 또는 해당 출구 이동을 다시 시도한다.

도둑이 아니면 `도둑만 자물쇠를 딸 수 있습니다.`가 나오고, 이미 잠겨 있지 않은 출구에는 `그것은 잠궈져 있지 않습니다.`가 나온다.

## 14. 물건과 돈 주기

같은 방에 접속 중인 다른 플레이어에게 소지품을 줄 수 있다. legacy 어순은 물건, 대상, 명령 순서다.

```text
작은 돌 Bob 줘
```

성공하면 내 화면에는 `당신은 Bob에게 작은 돌을 줍니다.`가 나오고, 상대에게도 누가 무엇을 줬는지 출력된다. 물건은 내 소지품에서 빠지고 상대 소지품으로 이동한다.

돈은 금액 뒤에 `냥`을 붙인다.

```text
1000냥 Bob 줘
```

성공하면 내 금이 줄고 상대 금이 늘어난다. 돈이 부족하면 `당신은 그만큼의 돈을 가지고 있지 않습니다.`가 나온다. 물건은 접속 중인 플레이어에게만 줄 수 있고, 돈은 같은 방의 NPC에게도 줄 수 있다.

## 15. 상점 NPC 살펴보기

기념품점의 NPC는 다음처럼 살펴볼 수 있었다.

```text
묘고부 봐
```

실제 출력:

```text
기념품을 팔아 생계를 이어가는 묘고부가 서 있습니다.
```

상점 물건 목록은 다음처럼 확인한다.

```text
품목
```

또는 NPC 이름을 앞에 붙여도 같은 목록이 나온다.

```text
묘고부 품목
```

실제 출력 요약:

```text
## 상품들

상품                               │                              가격
────────────────────────────────────┼───────────────────────────────────
다리보호대                         │                            10,000
머리보호대                         │                            10,000
목보호대                           │                            10,000
...
```

물건을 사려면 품목에 나온 물건 이름 뒤에 `사`를 붙인다.

```text
다리보호대 사
```

실제 출력:

```text
당신은 다리보호대를 10,000전에 샀습니다.
```

명령어를 앞에 두는 입력도 동작한다.

```text
사 다리보호대
```

구매한 물건은 `소지품`에서 확인할 수 있고, `정보`의 금이 가격만큼 줄어든다.
구매와 판매가 성공하면 플레이어 저장 큐에 올라가므로, 재시작 뒤에도 금과
소지품 변경이 보존되는 경로로 동작한다.

전당포나 수리점에서는 `가치`/`가격` 명령으로 가진 물건의 평가액도 확인할 수 있다.

```text
목검 가치
가치 목검
```

전당포에서는 가진 물건 이름 뒤에 `팔아`를 붙여 판매할 수 있다. 명령어를 앞에 두는 `팔아 목검` 형식도 동작한다.

```text
목검 팔아
팔아 목검
```

판매가는 전당포 평가액과 같고, 판매한 물건은 `소지품`에서 사라지며 `정보`의 금이 그만큼 늘어난다.

### 은행과 우체국

개인 은행 명령은 은행 방에서만 동작한다. 코드 기준으로 `잔액`은 잔고를
보여주고, `보관물`은 은행 보관함의 물건 목록을 보여준다. `5000냥 입금`,
`5000냥 출금`, `모두 입금`, `모두 출금`은 금을 옮기고, `<물건> 보관물`은
가진 물건을 은행 보관함에 넣는다. `<물건> 받아` 또는 `모두 받아`는 은행
보관함에서 소지품으로 꺼낸다.

```text
잔액
보관물
5000냥 입금
5000냥 출금
청동검 보관물
청동검 받아
```

은행이 아닌 방에서는 `은행에서만 가능합니다.`가 나온다. 개인 은행 변경은
플레이어와 은행 저장 큐에 올라간다. `받아`/`모두 받아`는 C 방식의 소지
개수와 무게 제한을 검사하며, `모두 받아` 중 돈 오브젝트는 소지품이 아니라
금액으로 합산된다. 패거리 금고/창고 방에서는 `기부`, `인출`,
`기증품`, `<물건> 기증`, `<물건> 반출`도 동작한다. 패거리 금고 입출금은
만냥 단위, 1000만냥 거래 한도, 10억 총액 제한, 큰 거래 방송, 저장 경로가
연결되어 있고, 창고 보관/반출은 물건 종류, 컨테이너, 개수, 소지 무게 제한을
검사한다. 재시작 smoke 증거는 별도 확인 대상으로 남는다.

우체국 명령도 우체국 방에서만 동작한다. `Bob 편지보내기` 또는
`편지보내기 Bob`은 편지 입력 모드로 들어가며, 줄 처음에 `.`을 입력하면
보낸다. `편지받기`는 내 편지 파일을 읽고, 긴 편지는 이어 읽기 프롬프트를
쓴다. `편지삭제`는 받은 편지 파일을 삭제한다.

```text
Bob 편지보내기
편지받기
편지삭제
```

우편은 legacy `post/` 파일을 직접 만들거나 삭제하므로 운영 smoke에서는
제외한다.

## 16. 훔치기, 검색, 숨기, 사용, 도망

`훔쳐`는 도둑 계열이 같은 방의 NPC나 플레이어가 가진 물건을 빼앗으려 할 때 쓴다. 어순은 `물건 대상 훔쳐`다.

```text
목검 생쥐 훔쳐
돈 상인 훔쳐
```

대상이 빠지면 `누구한테서 훔치려구요?`, 물건이 빠지면
`무엇을 훔치려구요?`가 나온다. 도둑 계열이 아니면
`도둑만 훔칠수 있습니다.`가 나오며, 눈이 멀어 있으면
`당신은 눈이 멀어 훔칠 수 없습니다.`가 나온다. 훔칠 수 없는 대상은
`당신은 그에게서 훔칠 수 없습니다.`, 대상이 가진 물건이 아니면
`그는 그런 물건을 갖고 있지 않습니다.`가 나온다. 성공하면 일반 물건은
내 소지품으로 옮겨지고 `훔쳤습니다.`가 나온다. 돈 물건은 돈으로
더해지며 `당신은 120냥을 훔쳐 220냥을 갖고 있습니다.`처럼 훔친 액수와
새 보유액이 나온다. 실패하면 `실패하였습니다.`가 나오고, NPC에게
실패하면 이어서 `그가 당신을 공격합니다.`가 나올 수 있다. 훔치기는
5초 대기 시간이 있고, 숨어 있던 상태가 풀린다. 투명 상태로 훔치기를
시도하면 대기 시간이 지난 실제 시도 시점에 모습을 드러낸다.
현재 Go 서버 기준으로 플레이어에게 훔치기를 실패하면 피해자 본인에게
`Alice님이 당신에게서 목걸이를 훔치려고 합니다.` 같은 직접 알림이
전송된다. 같은 방의 다른 접속자에게는 훔치기 시도 알림이 보이지만,
피해자에게는 별도의 피해자용 문장이 간다. 플레이어에게서 훔치기를
성공하면 훔친 사람에게 `plykl` 런타임 대기 시간이 7일에서 10일 사이로
설정된다. 이 값은 현재 실행 중인 Go 서버의 메모리 상태이며, 성공
메시지에는 별도로 표시되지 않는다. 훔치기 명령 자체가 바로 확인하는
대기 시간은 별도의 5초 `steal` 대기 시간이다.

`때려` 또는 `공격`은 같은 방에 보이는 NPC를 공격한다. 기본 어순은 `대상 때려`이고, 현재 포팅 서버는 `때려 대상`도 받아준다.

```text
생쥐 때려
때려 생쥐
```

공격하면 먼저 명중 판정을 한다. 현재 Go 서버의 기본 명중식은
`1~30 난수 >= 공격자 thaco - 대상 armor/10`이다. 빗나가면 `당신의
공격은 빗나갔습니다.`가 나오고 체력은 줄지 않는다. 맞으면 `당신은
생쥐에게 4만큼의 피해를 주었습니다.`처럼 실제 적용 피해량이 나온다.

피해량은 장비와 능력치에 따라 정해진다. 무장한 무기가 있으면 무기의
`nDice`, `sDice`, `pDice`로 주사위를 굴린 값에 힘 보너스와 무기 숙련
보너스를 더한다. 무기 숙련 보너스는 해당 무기 타입의 숙련도 값을 10으로
나눈 몫이다. 무기가 없고 공격자 자체 피해 주사위가 있으면 그 주사위와
힘 보너스를 쓴다. 둘 다 없으면 공격자 레벨만큼, 레벨도 없으면 1 피해를
준다. 최종 피해가 1보다 작아지는 경우에도 최소 1 피해로 보정된다.
공격자에게 `PUPDMG`/`upDamage` 계열 상태가 있으면 클래스와 레벨 조건에
따라 한 번의 명령에서 추가 공격이 발생할 수 있다.

처치하면 `생쥐가 쓰러졌습니다.`가 나오고, 대상은 방에서 사라진다. 처치된
몬스터가 가진 물건은 방에 떨어지고, 보유 금액은 `5냥` 같은 돈 물건으로
떨어진다. 교역품 보유 몬스터는 들고 있던 물건을 방에 떨어뜨리지 않고
삭제하지만, 보유 금액은 돈 물건으로 떨어진다.

전투 기여자가 기록된 경우 플레이어 공격자만 보상을 받는다. NPC가
몬스터에게 피해를 줬더라도 NPC는 경험치와 성향 보상을 받지 않는다.
경험치는 `몬스터 경험치 * 실제 누적 피해 / 몬스터 최대 체력`으로
계산되고, 한 공격자의 기본 경험치는 몬스터 경험치를 넘지 않는다. 보상
경험치를 받으면 현재 들고 있는 무기의 타입에 맞는 숙련도도 기본 경험치와
같은 양만큼 오른다.

처치 순간 공격자가 그룹에 속해 있으면 그룹 보너스가 기본 경험치에
추가된다. 리더는 `기본 경험치 / 5 * (팔로워 수 + 1)`, 팔로워는
`기본 경험치 / 4 * 팔로워 수`를 더 받는다. 기본 경험치가 몬스터 경험치와
같거나, 보너스 적용 전 계산이 몬스터 경험치의 두 배를 넘는 경우에는
추가 보너스가 기본 경험치와 같은 값으로 제한된다. 성향은 피해 비율로
나누지 않고, 기록된 플레이어 공격자마다 `현재 성향 - 몬스터 성향/5`로
조정되며 -1000에서 1000 사이로 제한된다. 보상 결과는 처치 순간 별도
메시지로 요약되지 않으므로, 경험치와 성향 변화는 `정보`나 `점수`로 다시
확인해야 한다.

플레이어를 공격하려고 하면 먼저 안전지대, 대련장, 혼돈 상태, 패거리
전쟁 여부를 검사한다. C 기준으로 공격할 수 없는 상황이면 `이 곳에서는
싸울 수 없습니다.`, `당신은 선하다는걸 아세요.`, `그 사용자는 선해서
공격할 수 없습니다.` 같은 거절 메시지가 나온다. 조건을 통과하면 연결된
플레이어 creature HP에 최소 피해 처리가 적용된다. 다만 C의 PvP 사망
부작용, 부활/시체/드롭/패널티, 피해자 직접 알림과 방 브로드캐스트는
아직 완성된 상태가 아니다. 공격할 수 없는 보호 대상은
`그 상대는 공격할 수 없습니다.`가 나온다. 자기 자신은 공격할 수 없다.

`차기`는 권법가 계열 전투 기술로 연결되어 있다. 대상이 맞으면 일반
공격과 별도로 발차기 피해가 들어가며, 실패하면 실패 메시지만 나온다.
플레이어 대상은 `때려`와 같은 안전지대/혼돈/패거리 전쟁 검사를 거친다.

`추적`은 레인저 계열 조회 명령으로 연결되어 있다. 대상 없이 쓰면 방에
남은 legacy track 방향을 보여주고, 대상을 붙이면 현재 방 또는 인접한
열린 출구 너머에서 보이는 대상의 흔적 방향을 알려준다. 눈이 멀었거나
레인저 계열이 아니면 거절된다.

`검색` 또는 `찾아`는 현재 방에서 숨겨진 출구, 물건, 숨어 있는 대상을 찾아본다.

```text
검색
찾아
```

아무것도 못 찾으면 `당신은 아무것도 찾지 못했습니다.`가 나온다. 너무 빨리 다시 쓰면 잠시 기다리라는 메시지가 나오고, 검색을 시도하면 내 `숨겨` 상태가 풀린다.

`숨겨` 또는 `숨어`는 대상 없이 쓰면 내가 숨고, 물건 이름을 붙이면 방에 있는 물건을 숨기려 한다.

```text
숨겨
숨어
작은 돌 숨겨
```

내가 숨을 때는 `당신은 애써 숨어보려고 합니다.`가 먼저 나오고, 성공하면 `당신은 성공적으로 숨었습니다.`가 나온다. 물건 숨기기는 `당신은 그것을 숨겨보려고 합니다.`가 나오며, 성공하면 `당신은 성공적으로 숨겼습니다.`가 나온다. 방에 없는 물건은 `그런것은 여기 없어요.`, 숨길 수 없는 물건은 `당신은 그것을 숨길 수 없습니다.`가 나온다. 연속 사용에는 대기 시간이 있다.

`사용`은 물건 종류에 따라 적절한 행동으로 이어지는 초보자용 단축 명령이다.

```text
목검 사용
갑옷 사용
횃불 사용
치료약 사용
귀환 사용
마법완드 사용
열쇠 서 사용
```

무기는 `무장`, 방어구는 `입어`, 물약은 `마셔`, 광원은 `잡아`와 같은
처리를 한다. 주문서는 `읽어`, 완드/지팡이는 `zap`과 같은 경로로
이어진다. `special=SP_MAPSC` 지도/문서 아이템은 `읽어`와 `사용`에서
`objmon/<아이템명>` 파일을 출력한다. 긴 파일은 20줄 단위로 나뉘어
계속 읽을 수 있고, `.`으로 중단할 수 있다. `special=SP_COMBO` 조합자물쇠는
숫자 입력을 누적해 `Click.`을 출력하고, 정답을 모두 맞히면 물건 설정에
따라 방 출구의 잠김과 닫힘을 풀 수 있다. 틀린 조합을 끝까지 입력하면
피해를 입을 수 있다. C의 별도 입력 모드와 완전히 같은지는 아직 갭으로
남아 있으므로, 초보자는 특수 퍼즐 물건 정도로 이해하면 된다.
따라서 `귀환 사용`처럼 주문서를 `사용`해도 성공 조건을
통과하면 주문서가 소모되고, `마법완드 사용`은 완드 판정, 회수, 방의
마법 가능 여부를 `zap`과 같은 방식으로 검사한다. 다만 `special=3`,
`special=SP_WAR`, `SP_WAR=1`처럼 패거리 전쟁용 특수 물건으로 표시된
아이템은 일반 사용 경로로 보내지지 않고 C `special_obj` 기본값인
`아무것도 없습니다.`를 출력한다. 주문서 소모나 완드 회수 소모도 일어나지 않는다.
직접 주문을 외우는 `주문`/`cast` 명령도 같은 기본 주문 효과 디스패처에
연결되어 있다. 숫자 주문과 이름 주문을 모두 받아들이며, 예를 들어
`주문 1 상인`, `상인 회복 주문`, `주문감지술 주문` 같은 형태를 처리한다.
주문서는 `배워` 또는 `연마`로 학습할 수 있고, 성공하면 주문서가 소모된다.
`활보법`과 `신원법`도 직업/상태/대기시간 조건을 검사해 런타임 상태를
바꾸는 명령으로 연결되어 있다.
현재 Go 서버의 기본 주문 효과 디스패처는 light, 해독, 소량 치유, 일부
피해 주문, 축복, 보호, 투명, 투명감지, 마법감지 태그, befuddle, full
heal, 실명회복, 질병회복, 공중부양/비행/저항 계열 태그, 성향감지,
수중호흡, earth shield, 공포/실명/침묵 상태, 귀환을 연결한다. 마법감지 상태에서는 `봐`/`보다`로 방이나
물건을 볼 때 마법 물건 표시가 붙고, 도둑/관리자급 `감정`에는 마법 세부
정보가 추가될 수 있다. `소지품` 목록에도 소지품의 마법감지 전용 표시가 붙을 수 있다.
주문서와 완드는 같은 디스패처를 쓰므로 아직 연결되지 않은 주문 번호는
발동 효과가 없어서 성공 처리와 회수 소모까지 가지 않는다. 열쇠는
`열쇠 서 사용`처럼 `열쇠 출구 사용` 형태로 잠긴 출구를 푸는 동작으로
이어진다. 맞는 열쇠이고 사용 회수가 남아 있으면 출구의 잠김이 풀리고
열쇠 사용 회수가 줄어든다. 열쇠가 아니면
`그것은 열쇠가 아닙니다.`, 맞지 않으면 `열쇠가 맞지 않습니다.`,
회수가 없으면 열쇠가 부서졌다는 메시지가 나온다. `서 열쇠 풀어`처럼
직접 `풀어`를 쓰는 형식도 같은 해제 동작이다. 반대로 `서 열쇠 잠궈`는
닫혀 있고 잠글 수 있는 출구를 잠그며, 현재 코드 기준으로 열쇠 사용
회수는 줄지 않는다.

대상이 없거나 찾을 수 없으면 `무엇을 사용하시려구요?`, 사용법이
정해지지 않은 물건이면 `그것을 어떻게 사용하시려구요?`가 나온다.
바닥 물건은 사용 가능 표시가 있는 일부 물건만 바로 `사용`할 수 있다.
`사용`을 시도하면 숨어 있던 상태가 풀린다.

물약도 마법 물건 제한을 받는다. 물약이 아니면 `이것은 먹는 물건이
아닙니다.`가 나오고, 사용 회수가 없거나 마력이 0이면 `아무것도
들어있지 않습니다.`가 나온다. 물약 금지 방이나 대련장에서는 먹을 수
없고, 선/악 전용 물약과 직업 제한 물약은 주문서/완드와 같은 성향 및
직업 제한을 검사한다. 제한에 걸린 선/악 전용 물약은 입안에서 타오르며
떨어지고, 직업 제한에 걸리면 먹을 수 없다는 메시지가 나온다. 현재 기본
물약 효과도 주문서/완드와 같은 저위험 효과 디스패처를 일부 사용하므로,
해독 물약처럼 연결된 주문 번호는 상태 태그를 실제로 바꿀 수 있다.
다만 일반 물약은 아직 연결되지 않은 주문 번호도 마신 처리와 회수 소모가
진행될 수 있다.
`OSPECI`/specialItem 물약은 일반 회수/마력 검사보다 먼저 처리되고, 혼돈
토글, 능력치 상승, 안전한 방 이동, 독 부여처럼 현재 모델에 맞는 `pdice`
효과는 동작한다. 플레이 시간 증감과 C `load_rom`의 세부 부작용은 아직
남아 있다.

`읽어`는 게시판 읽기와 주문서 읽기 양쪽으로 연결되어 있다.
현재 방에 게시판이 있고 `읽어`만 입력하면 게시판 목록이 나오며,
`게시판 읽어`는 글 목록을, `게시판 2 읽어`는 해당 글 본문을 보여준다.
주문서 이름을 대상으로 주면 인벤토리나 장비에서 주문서를 찾아 읽는다.
주문서가 아니면 `이것은 스크롤이 아닙니다.`가 나온다. 눈이 멀어 있으면
읽을 수 없고, 주문서의 `nDice` 요구 레벨이 내 레벨보다 높아도 읽을 수
없다. 선 전용 주문서는 성향이 -100보다 낮으면 손에서 떨어지고, 악 전용
주문서는 성향이 100보다 높으면 손에서 떨어진다. 직업 제한 주문서는 내
직업 플래그가 맞아야 읽을 수 있다. 사용 회수가 없거나 마력이 0이면
`아무런 일도 일어나지 않았습니다.`가 나오고, 마법 금지 방에서도 같은
결과가 나온다.

조건을 통과하면 현재 Go 서버 기본 핸들러는 일부 저위험 주문 효과를
처리한 뒤 사용 문구를 출력하고 주문서를 소모한다. 현재 기본 효과는
light, 소량 치유, 일부 피해 주문, 해독, 축복, 보호, 투명, 투명감지,
마법감지, befuddle, 귀환, full heal, 공중부양/비행/저항 계열 태그,
성향감지, 수중호흡, earth shield, 질병/실명 회복, 공포/실명/침묵
상태다. 저주, 소환, 복잡한 이동, 지속시간/저항 판정, 광역 효과 같은
주문표 효과는 아직 대부분 연결되지 않았다.
주문서 대상 `읽어`와 주문서 `사용`은 실제 소지품을 바꾸므로 운영
smoke에서는 제외한다.

```text
게시판 읽어
게시판 2 읽어
귀환 읽어
```

`zap`은 완드/지팡이류 아이템을 겨냥하는 명령으로 연결되어 있다.
대상 아이템이 완드인지, 사용 회수가 남았는지, 방에서 마법을 쓸 수
있는지를 검사한다. 눈이 멀면 `아무 것도 보이지 않습니다!`가 나오고,
완드가 아니면 `완드나 지팡이가 아닙니다.`가 나온다. 사용 회수가 0이면
`모두 사용되었습니다.`, 마력이 0이거나 마법 금지 방이면 `아무 일도
일어나지 않습니다.`가 나온다. 완드도 주문서와 같은 성향 제한과 직업 제한을
검사한다. 제한에 걸린 선/악 전용 완드는 손에서 떨어지고, 직업 제한에
걸리면 사용할 수 없다는 메시지가 나온다.

현재 Go 서버 기본 핸들러는 일부 피해, 상태, 보조 완드 효과만 처리한다.
해독, 투명감지, 귀환이나 일부 피해 주문처럼 연결된 효과는 성공 후
`useOutput`을 출력하고 완드 회수를 줄인다. 아직 연결되지 않은 주문
번호는 발동 효과와 회수 소모 없이 끝난다.

`도망` 또는 `도`는 같은 방에 보이는 적대 NPC가 있을 때 임의의 보이는 출구로 달아난다.

```text
도망
도
```

도망칠 대상이 없으면 `누구에게서 도망가시려구요?`가 나온다. 실패하면 `당신은 겁에 질려 다리가 떨어지지 않습니다!`가 나오고 이동하지 않는다. 성공하면 `당신은 줄행랑을 칩니다.`가 나오며 새 방 설명이 이어진다. 닫힌 문, 숨겨진 출구, 보이지 않는 출구, 입장 제한이 걸린 출구로는 도망치지 않는다. 성기사 계열이고 레벨이 20보다 높으면 성공한 도망에 경험치 패널티가 붙는다. 현재 계산식은 `((레벨 + 3) / 4) * 10`이며, 보유 경험치보다 많이 잃지는 않는다.

## 17. 막혔을 때의 기본 순서

명령이 막히면 다음 순서로 확인한다.

1. `봐`로 현재 방 설명과 출구를 다시 본다.
2. 출구가 방향이면 `북`, `남`, `동`, `서`, `위`, `밑`, `밖`처럼 그대로 입력한다.
3. 출구가 이름이면 `백화점 가`, `기념품점 가`처럼 `이름 가`를 시도한다.
4. 잠긴 출구는 열쇠가 있으면 `열쇠 서 사용` 또는 `서 열쇠 풀어`를 시도한다.
5. 물건이나 NPC는 `사물함 봐` 또는 `봐 사물함`처럼 살펴본다.
6. 말하기는 `안녕하세요 말` 또는 `말 안녕하세요`를 쓴다.
7. 귓속말은 `Bob 안녕하세요 이야기`, 전체 잡담은 `모두 안녕 잡담`처럼 쓴다.
8. 같이 움직일 때는 같은 방에서 `Bob 따라`, 확인은 `그룹`, 그룹 안 대화는 `모두 준비됐나요 그룹말`을 쓴다.
9. 패거리 소속이면 `패거리누구`로 접속 중인 패거리원을 보고, `모두 준비됐나요 패거리말`로 패거리 안에 말한다.
10. 같은 방 접속자에게 물건이나 돈을 줄 때는 `작은 돌 Bob 줘`, `1000냥 Bob 줘`처럼 쓴다.
11. 감정 표현은 `미소`, `계석치무 안녕`처럼 쓴다.
12. NPC와 이야기할 때는 `계석치무 대화`, 주제를 물어볼 때는 `계석치무 임무 대화`처럼 쓴다.
13. 가진 무기나 방어구는 `목검 비교`로 사용 가능 레벨을 확인한다.
14. 도둑 계열이면 `목검 감정`으로 물건의 세부 수치를 확인한다.
15. 숨겨진 것을 찾을 때는 `검색`, 내가 숨거나 방 물건을 숨길 때는 `숨겨`를 쓴다.
16. 도둑 계열이고 열쇠가 없으면 `서 따`처럼 잠금 따기를 시도할 수 있다.
17. 무기, 방어구, 물약, 광원은 `목검 사용`처럼 `사용`으로 처리할 수 있다.
18. 전투 중 위험하면 `도망`을 쓰되, 실패하거나 갈 수 있는 출구가 없을 수 있다.
19. 현재 접속자는 `누구`, 소지품은 `소지`, 전체 도움말은 `도움말`, 초보 안내는 `환영`으로 확인한다.
20. 길을 잃었으면 `귀환`으로 `생명의 나무`로 돌아갈 수 있다.

## 안전한 스모크 명령

실서버 접속 상태에서는 조회형 명령만 smoke로 쓴다. 성공했을 때 방,
소지품, 장비, 돈, 출구, 체력/경험치, 숨김/투명, 대기 시간, 게시판
내용을 바꾸는 명령은 운영 smoke에서 제외한다. `검색`은 대기 시간과
숨김 해제가 있고, `숨겨`, `사용`, `주문`, `배워`, `연마`, `활보법`,
`신원법`, `기공집결`, `살기충전`, `흡성대법`, `긁어`, `영자팔법`, `나한진`,
`변수나한권`, `타구봉법`, `반탄강기`, `분신술`, `창격술`, `확인사살`,
`실명해소술`, `최루탄`, 주문서 대상 `읽어`, `zap`, `도망`, `훔쳐`, `때려`, `공격`, `차기`,
`가져`, `버려`, `줘`, `사`, `팔아`, `귀환`,
은행 입출금/보관/받아, 우편 쓰기/삭제, 게시판 쓰기/삭제, 대상을 붙인
`설정`/`해제`, 이동 명령은 상태를 바꾸거나 바꿀 수 있으므로 운영
smoke에서 제외한다.

```text
도움말

봐

게시판 봐
계석치무 봐
```

위험도 구분:

- 낮음: 게시판이 없는 방의 `읽어` 단독은 대상 없음 메시지만 내는
  실패 명령이다. 게시판이 있는 방의 `읽어`, `게시판 읽어`, `게시판 2 읽어`는
  게시판 파일을 읽어 출력만 만든다.
- 높음: 주문서 대상 `읽어`와 주문서 `사용`은 성공 시 주문서를
  소모한다. 초보자 계정의 실제 소지품을 잃을 수 있으므로 smoke에서
  제외한다.
- 높음: `사용`은 물건 종류에 따라 장비 착용, 무장, 물약 마시기, 광원
  잡기, 주문서 읽기, 완드 겨냥, 열쇠 사용으로 라우팅된다. 어떤 물건을
  대상으로 하느냐에 따라 상태 변경이 달라지므로 운영 smoke에서는
  대상 없는 실패 메시지 확인 용도로도 반복하지 않는다.
- 높음: `zap`은 일부 저위험 완드 효과가 연결되어 있어 대상 완드에 따라
  회수 소모와 효과 발동이 가능하다. 실제 완드를 대상으로 하는 smoke에서는
  제외한다.
- 높음: `주문`/`cast`는 MP, 상태 태그, 체력, 이동 같은 런타임 상태를
  바꿀 수 있다. 대상 없는 실패 메시지 확인 외에는 운영 smoke에서
  제외한다.
- 높음: `배워`/`연마`, `활보법`, `신원법`은 주문서 소모, 능력치, 상태
  태그, 대기시간을 바꿀 수 있으므로 운영 smoke에서 제외한다.
- 높음: `열쇠 서 사용`은 출구 잠김을 풀고 열쇠 사용 회수를 줄이는
  상태 변경 명령이다. `서 열쇠 풀어`, `서 열쇠 잠궈`도 출구 상태를
  바꾸므로 운영 smoke에서 제외한다.
- 높음: `훔쳐`는 성공하면 물건/돈을 옮기고, 실패해도 5초 대기 시간,
  숨김 해제, 투명 해제, 플레이어 피해자 알림 같은 런타임 상태와
  메시지를 만들 수 있다. 플레이어 피해자를 상대로 성공하면 `plykl`
  런타임 대기 시간도 설정된다.
- 높음: `때려`/`공격`/`차기`는 체력, 몬스터 생존 여부, 방 드롭, 경험치와
  성향을 바꿀 수 있다. `도망`도 이동과 경험치 패널티가 생길 수 있어
  smoke에서 제외한다.
- 높음: `써`/`글삭제`, `편지보내기`/`편지삭제`는 legacy 파일을 직접
  만들거나 바꾼다. `편지받기`와 게시판 읽기는 조회지만, 쓰기/삭제
  명령은 운영 smoke에서 제외한다.
- 높음: `입금`/`출금`/`받아`, 대상이 붙은 `보관물`은 금이나 물건을
  이동한다. 개인 은행은 JSON sidecar 저장 경로가 있지만, smoke에서는
  원본 계정의 금과 물건을 바꾸지 않도록 조회형 `잔액`/`보관물`만 쓴다.
- 중간: 대상 없는 `설정`은 조회지만, 대상을 붙인 `설정`/`해제`는
  플레이어 설정 태그를 바꾼다.

터미널에서 확인할 때는 DB 쓰기, 파일 생성, 장기 리슨을 피하는 명령만 사용한다.

```text
go test ./internal/engine/command
go run ./cmd/muhan-server -root . -dry-run
go run ./cmd/muhan-server -root . -validate
go run ./cmd/muhan-cmdlist -root .
go run ./cmd/muhan-boardmap -root . -json
go run ./cmd/muhan-dbimport -root . -run-id smoke-doc -json
```

주의: `./scripts/db-smoke-postgres.sh`와 `go run ./cmd/muhan-dbimport ... -execute`는 PostgreSQL에 실제 import run을 쓴다. 운영 DB나 보존해야 하는 로컬 DB에서는 smoke로 쓰지 않는다.

### 포팅 검증 smoke 기준

반복 검증은 [porting-smoke-checklist.md](/workspace/muhan/docs/porting-smoke-checklist.md)를 기준으로 한다. 자동화된 기본 검증은 다음처럼 실행한다.

```text
scripts/porting-smoke.sh
scripts/porting-smoke.sh --runtime --skip-go-test
scripts/porting-smoke.sh --all --copy-root
scripts/porting-smoke.sh --live-gap-report --no-preflight --no-sidecar
```

첫 번째 명령은 빌드, 테스트, validate, command list, sidecar dry-run을 실행하고 `/tmp/muhan-porting-smoke.*` 아래에 로그와 `gap-report.txt`를 남긴다. `--runtime`은 `muhan-server`와 `muhan-cli` tmux 세션을 띄운 뒤 `인제로` / `1234`로 접속해서 읽기 중심 명령을 보낸다. 원본 데이터에 sidecar 저장이나 shutdown flush 흔적을 남기면 안 되는 확인은 `--copy-root`를 붙여 disposable root에서 실행한다.

남은 C parity 갭을 고를 때는 먼저 다음처럼 비파괴 evidence 보고서를 만든다.

```text
scripts/porting-smoke.sh --runtime --copy-root --live-gap-report --skip-go-test --no-sidecar --leave-sessions --port 4042
```

이 실행은 복사된 root로 서버를 띄우고 `live-gap-report.txt`에 실제 talk
ACTION/GIVE/ATTACK/CAST 후보, family 후보, room/object fixture 수를 적는다.
이후 `tmux attach -t muhan-cli`로 들어가서 하나의 갭만 골라 확인하고,
`tmux capture-pane -pt muhan-cli -S -500`으로 전후 화면을 남긴다. 전투,
trap, talk GIVE, 패거리 추방, class 변경, ORENCH 생성/드롭처럼 상태를
바꾸는 검증은 반드시 이렇게 복사 root에서만 한다.

fixture 이름이나 본문이 깨져 보이면 데이터가 아직 legacy byte 상태일 수
있다. 원본 데이터를 바로 고치지 말고, 읽기 전용 UTF-8 audit을 먼저 남긴다.

```text
scripts/utf8-audit.sh --root /workspace/muhan --output /tmp/muhan-utf8-audit.txt
```

이 보고서는 `family_member_N`, `board`, `room/json`, `player/json`,
`player/bank/json`의 파일명/내용이 UTF-8인지, CP949 계열 legacy text인지,
binary record인지 분류한다. 자세한 변환 순서는
[utf8-migration-plan.md](/workspace/muhan/docs/utf8-migration-plan.md)를 따른다.

가족 명단처럼 텍스트 파일만 먼저 변환 후보를 보려면 dry-run으로 실행한다.

```text
scripts/utf8-convert-text.sh --root /workspace/muhan --include family --manifest /tmp/muhan-family-convert.manifest
scripts/utf8-rename-paths.sh --root /workspace/muhan --include all --manifest /tmp/muhan-path-rename.manifest
```

이 명령들은 기본적으로 파일을 고치지 않는다. 실제 `--write`는 복사한 root에서
먼저 실행하고, `validate`와 접속 smoke가 통과할 때만 원본 반영을 검토한다.

초보 플레이어 역할에서는 다음 명령까지만 기본 smoke로 사용한다.

```text

장비
소지
게시판
계석치무 봐
```

`묘사`, `줄`, `암호`, `제련`, `무기만들기`, `직업전환`, `자살`, `때려`,
`훔쳐`, `사용`, `주문`, `입금`, `출금`, 게시판 쓰기/삭제, 우편 삭제는

legacy 명령표의 handler key는 현재 Go 서버에 모두 연결되어 있다. 다만
일부 명령은 안전한 최소 동작이나 hook 기반 구현이라 C와 세부 side effect가

`기부`/`기증`/`기증품`/`인출`/`반출`은 현재 핸들러가 연결되어 있다.
패거리 공지는 `패거리공지`로 읽고, `패거리공지 a`로 한 줄씩 추가한 뒤
문장 처음에 `.`을 입력해 저장을 끝낸다. `패거리공지 d`는 패거리 두목만
공지 파일을 삭제할 수 있다. 패거리원 직접 편집은 아직 플레이 기능으로
이어지지 않는다.
talk 파일의 `CAST`와 `ATTACK`은 실행 순서와 기본 side effect가 연결되어
있고 단위 테스트도 있다. 다만 실제 talk 파일을 통한 전투 tick, 주문 엔진
전체, 재시작 후 상태까지의 증거는 아직 `--live-gap-report` 후보를 골라
복사 root에서 transcript로 남겨야 한다.
새로 연결된 `투표`, `결혼`/`초대`, `무적수련`, `메모`, `칭호`, 무적 기술
계열은 안전한 런타임 동작부터 들어간 상태라 legacy 파일 저장, 상태 만료,
확인 프롬프트, 적대 목록, 자동 도주/사망 부작용, 세부 제한 조건은 아직
C와 다를 수 있다.

### 명령 어순이 초보자에게 헷갈릴 수 있다

legacy 방식은 `대상 명령` 어순이지만, 현재 포팅 서버는 일부 기본 명령에 대해 `명령 대상` 어순도 받아준다.

```text
봐 사물함
봐 게시판
봐 계석치무
말 안녕하세요
외쳐 도와줘
이야기 Bob 안녕하세요
잡담 모두 안녕하세요
그룹말 모두 준비됐나요
패거리말 모두 준비됐나요
```

다음 legacy 어순도 계속 동작한다.

```text
사물함 봐
게시판 봐
계석치무 봐
안녕하세요 말
```

도움말에는 앞으로 두 어순을 모두 안내하면 좋다.

### 이름 있는 출구는 그대로 입력하면 실패한다

광장 출구 목록에 `백화점`이 보이지만 `백화점`만 입력하면 실패했다.

재현:

```text
백화점
```

출력:

```text
무슨 말인지 모르겠습니다.
```

성공 명령:

```text
백화점 가
```

출구 목록에 방향과 이름 있는 출구가 함께 나오므로, 초보자는 둘 다 그대로 입력해도 될 것처럼 느낄 수 있다.

### NPC 대화 주의

광장에서 보이는 NPC 계석치무는 `계석치무 대화`로 기본 대사를 들을 수 있고, `계석치무 임무 대화`처럼 주제를 붙이면 legacy talk 파일의 해당 응답을 읽는다.

성공 명령:

```text
계석치무 대화
계석치무 임무 대화
```

출력 요약:

```text
계석치무가 당신에게 "또 다른 '멍청이'구만 .. 암튼 중간계에 온것을 환영하네."라고 이야기합니다.
계석치무가 당신에게 "어느 멍청한 배달부가 내 타버린 두루마기를 수련장에서 잃어버렸네 ..."라고 이야기합니다.
```

talk 파일에 들어 있는 `ACTION` 부가 지시는 대사 출력 직후 NPC의 감정표현으로 실행된다. `GIVE` 지시는 legacy `atoi`식 숫자 보상, nested object clone, 중복 보상 거절, 실패 rollback 경로가 연결되어 있다. `ATTACK` 지시는 NPC가 플레이어를 적으로 등록하고 다음 전투 tick으로 이어질 수 있게 준비한다. `CAST` 지시는 runtime hook과 기본 주문 fallback을 통한다.
현재 포팅 코드는 talk `ACTION/GIVE/ATTACK/CAST`의 실행 순서, 중복 보상 거절,
무게/퀘스트 실패 rollback, enemy primer, runtime CAST hook을 단위 테스트로
검증한다. 그래도 초보 플레이어가 실제 데이터를 검증할 때는
`--live-gap-report`가 제시한 후보 talk 파일을 보고, 복사 root에서 대화 전
`정보` 후 엔터/`소지`/`봐`, 대화 후 `정보` 후 엔터/`소지`/`봐`, 재시작 후 같은 명령을
차례대로 비교해야 한다. 원본 계정에서 보상 대화나 공격 대화는 반복하지
않는다.

### 상점 구매와 전당포 판매 흐름

백화점 로비에서 `기념품점 가`로 상점에 들어갈 수 있고, NPC `묘고부`도 보인다. 품목 확인과 구매가 가능하다.

성공 명령:

```text
품목
묘고부 품목
```

출력 요약:

```text
## 상품들

상품                               │                              가격
────────────────────────────────────┼───────────────────────────────────
다리보호대                         │                            10,000
   ...
```

구매는 `다리보호대 사` 또는 `사 다리보호대` 형식으로 동작한다. 구매 전에
소지 개수와 무게 제한을 검사하므로 너무 무겁거나 이미 너무 많이 들고 있으면
돈과 물건이 변하지 않는다. `가치`/`가격`은 전당포나 수리점에서 가진 물건을
평가하는 용도로 구현되어 있고, 전당포에서는 `다리보호대 팔아` 또는
`팔아 다리보호대` 형식으로 판매할 수 있다. `purchaseItems` 몬스터의
`carry[]` 판매품도 legacy prototype clone으로 구입된다.

### 가져오기와 장비 착용 흐름이 불명확하다

사물함에는 `여포의 단도`가 보이지만 가져오기는 실패했다.

재현:

```text
여포의 단도 가져
```

출력:

```text
그런건 여기 없어요.
```

소지품에 `목검`이 있으면 다음 명령으로 무장할 수 있다.

성공 명령:

```text
목검 무장
```

출력:

```text
당신은 목검으로 전투태세를 취합니다.
```

확인:

```text
장비
```

출력:

```text
장비:
wield: 목검
```

`목검 입어`는 실패하는 것이 맞다. 목검은 입는 방어구가 아니라 `무장`하는 무기다. 아이템을 보고, 집고, 장비하는 기본 루프는 더 넓은 물건 종류로 계속 검증이 필요하다.

### 클라이언트 세션에 쉘 출력이 섞일 수 있다

tmux 캡처 기록에 `socat` 접속이 종료된 뒤 쉘 프롬프트와 `clear` 오타가 게임 화면 중간에 섞인 흔적이 있었다.

예:

```text
dh.kam@snowbox:/workspace/muhanclear
Command 'lear' not found
```

게임 프롬프트와 쉘 프롬프트가 붙어서 보이면 현재 입력이 게임으로 가는지 쉘로 가는지 초보자가 혼동할 수 있다. 접속이 끊겼을 때는 새 `socat` 명령으로 다시 연결해야 한다.
